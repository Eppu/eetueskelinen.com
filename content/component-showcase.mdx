## Text & Reading Aids

Most posts are composed of prose, but sometimes pure text needs to be broken up.

<BigQuote>
  A digital garden is a collection of evolving ideas that are inherently networked rather than strictly chronological.
</BigQuote>

Wait, what if the reader doesn't understand your perspective? Let them know it's fine! 

<Callout title="Wait, how is this different from a blog?" variant="info">
  While blogs are strictly time-bound streams of finished thoughts, digital gardens are atemporal webs of evolving ideas!
</Callout>

Or, if you need to go off on a really long tangent, use a Deep Dive:

<DeepDive title="Why I Chose Next.js App Router for this">
  The combination of `next-mdx-remote/rsc` and React Server Components means I can write these markdown files, compile them into HTML purely on the server, and only ship the tiny bits of JS necessary for interactivity (like the collapsible header up top!). No heavy client-side hydration for 90% of the text.
</DeepDive>

Occasionally, an aside note is perfect for a small caveat:

<AsideNote label="Author's Note">Like this one right here!</AsideNote>

## Code & Development

If you are writing about software, code blocks are non-negotiable. Standard markdown handles snippets fine:

```ts
function sum(a: number, b: number): number {
  return a + b;
}
```

But when you want to show shell commands, you need something punchier:

<TerminalWindow title="bash - eetueskelinen.com">
  <span className="text-yellowgreenlight">~</span> $ npm run dev <br />
  <span className="text-neutral-500">&gt; next dev --turbo</span><br />
  <span className="text-sky-400">ready</span> started server on 0.0.0.0:3001, url: http://localhost:3001
</TerminalWindow>
